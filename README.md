`debate-prep` 把通用 AI 变成一名可协作的华语辩论备赛教练：收到辩题后先自行诊断命题类型与比较结构，再陪你完成头脑风暴、统一框架、四个辩位的完整准备包与限时稿、模拟攻防、证据核查和赛后复盘。

它不会只给一篇“看起来像辩论稿”的长文，而是尽量建立一套全队能够调用、攻守可以回写、临场便于重组的备赛系统。

> [!NOTE]
> 本项目以 OpenAI Codex 为主要测试环境，采用开放的 Agent Skills 目录结构。其他支持 Agent Skills 的客户端也可尝试加载，但触发方式、文件权限和脚本执行体验可能不同。

## 核心能力

| 能力 | 具体行为 |
| --- | --- |
| 自动诊断辩题 | 区分政策行动、价值评价、事实描述、因果效果、责任归属、概念解释等命题类型，并单独识别 A/B 比较、趋势、必要性等论证结构 |
| 建立全队唯一口径 | 统一定义、论域、判准、论证义务、可让步边界、论证图和证据状态，减少各辩位互相冲突 |
| 四辩位双层准备 | 先为一二三四辩建立较完整、可检索的战术包，再压缩为符合赛制时间的上场稿 |
| 有限而充分的发散 | 覆盖主要论证空间，同时以功能去重；只有新增前提、机制、证据、反例、影响或战术用途的内容才保留 |
| 最强对手模拟 | 优先模拟对方最强合理版本，提供问题树、分支追问、接质短答、自由辩短打和条件式结辩模块 |
| 证据与逻辑核查 | 对数据、政策、法律和学术结论要求查源；未核验内容标记为 `【待核验】`，并检查论证链的断点 |
| 个性化训练 | 支持温和、标准、高压三种对抗强度，并根据用户明确提供的背景补充少量、可迁移的跨学科知识 |
| 历史材料利用 | 将用户提供的 DOCX、PDF、Markdown 和文本材料视为团队语料，提炼方法而不擅自推断作者身份或个人风格 |

## 设计取向

- **先诊断，后站队。** 用户不需要先判断这是“政策辩还是价值辩”。
- **比较不是独立题型。** 一个辩题可以同时是价值评价型和 A/B 比较型；skill 会分别诊断命题类型与论证结构。
- **先全队框架，后分辩位写稿。** 一辩与统一框架先固定公共前提，二三四辩再围绕同一论证图展开。
- **完整准备包不等于无限堆料。** 二三四辩在限时稿之外保留必要的分支准备，但控制数量并消除重复。
- **攻击论证，不攻击辩手。** 默认表达温和、清楚、逻辑优先，也可以按用户要求调整风格。
- **AI 不替代事实核验。** 精确数字、法律政策和时效性事实必须回到原始或权威来源确认。

## 快速开始

可以显式使用 `$debate-prep`，也可以直接提出匹配的备赛任务，让 Codex 自动选择该 skill。

### 从一条辩题开始

```text
使用 $debate-prep 分析辩题“人的成长更体现在接纳自己还是改变自己”。
我是反方，先给我辩题诊断卡，再陪我做双方公平脑暴，不要马上出完整稿。
```

### 准备指定辩位

```text
使用 $debate-prep。我们采用四人制，我打二辩，驳论 2 分 30 秒。
请先给我完整准备包，再压缩成限时稿；不要重复一辩已经承担的定义和框架。
```

### 进行模拟攻防

```text
使用 $debate-prep，以高压模式扮演对方三辩。
一次只问一个问题，根据我的回答继续追问；每轮先简评，再给一个更好的短答。
```

### 使用历史材料

```text
使用 $debate-prep 分析这个材料文件夹。
先建立索引，再区分可复用的方法、具体立场、待核验数据和可能的表达技法，不要推断个人风格。
```

### 建议提供的信息

只给辩题也可以开始。信息越完整，后续产物越贴近实际比赛：

| 优先级 | 信息 |
| --- | --- |
| 必要 | 辩题、持方 |
| 强烈建议 | 赛制、各环节时间、你的辩位、比赛日期 |
| 可选 | 证据规则、评委偏好、队友分工、已有材料、希望的训练强度 |

当缺失信息不会改变早期脑暴时，skill 会明确假设后继续；只有不同答案会实质改变框架时才会追问。

## 工作流

```mermaid
flowchart LR
    A[辩题诊断] --> B[双方公平脑暴]
    B --> C[统一框架与战场]
    C --> D[四辩位完整准备包]
    D --> E[压缩限时稿]
    E --> F[模拟攻防]
    F --> G[问题回写与复盘]
```

默认工作区只保留五份核心文件，避免团队材料被拆得过细：

| 文件 | 主要内容 |
| --- | --- |
| `01-一辩与全队框架.md` | 辩题诊断、统一口径、我方论证图、共享证据、可控靶点、一辩稿和接质边界 |
| `02-二辩准备.md` | 对方路线、预判驳论、我方深化、数据与因果拆解、接质短答和限时陈词 |
| `03-三辩攻防.md` | 对方最强理论、对方论证图、攻击矩阵、质询问题树、自由辩短打和小结 |
| `04-四辩准备.md` | 首质、全场战场表、条件式结辩模块、价值回扣和临场重组稿 |
| `05-模辩与复盘.md` | 模拟设置、逐轮记录、问题回写、跨学科拓展和赛后复盘 |

## 辅助脚本

脚本仅使用 Python 标准库，建议 Python 3.10 或更高版本。它们提供机械化辅助，不替代辩论判断。

### 创建五文件比赛工作区

```bash
python scripts/create_match_workspace.py \
  --motion "辩题" \
  --side "持方" \
  --output "比赛工作区"
```

如果目标目录已经存在同名模板文件，脚本默认拒绝覆盖。确认目标无误后才使用 `--force`。

### 审计备赛工作区

```bash
python scripts/audit_workspace.py "比赛工作区"
```

审计内容包括：未替换占位符、缺少核验标记的数字、跨文件重复、战术 ID 冲突以及限时稿的估算时长。可以通过 `--cpm` 调整每分钟口语单位估计，或使用 `--strict` 在出现警告时返回非零状态码。

### 为历史材料建立元数据索引

```bash
python scripts/index_materials.py "材料根目录" \
  --output "比赛工作区/material-index.md" \
  --exclude .agents
```

索引脚本支持 `.docx`、`.pdf`、`.md` 和 `.txt`，只生成元数据索引，不复制原始材料。PDF 默认只记录文件层面的元数据，不宣称已经理解全文。

## 项目结构

```text
debate-prep/
├── SKILL.md                    # 核心触发描述与执行流程
├── agents/openai.yaml          # Codex 界面元数据
├── references/                 # 按需加载的诊断、辩位、证据和训练方法

├── scripts/                    # 创建、索引和审计脚本
├── README.md                   # 面向使用者的项目说明
└── LICENSE                     # MIT License
```

这种结构遵循渐进式加载：客户端先看到 skill 的名称与描述，触发后读取 `SKILL.md`，只有当前任务需要时再加载具体参考文件或运行脚本。

## 使用边界

- 本仓库不包含创建者或任何辩论队的历史赛稿、姓名、联系方式和私人材料。
- 用户自行提供的材料可能受版权、保密或赛事规则约束；公开分享前请自行确认授权并完成脱敏。
- 模型生成的案例、数字、引文和政策表述可能出错或过时。正式比赛前应核查原始来源。
- 机械审计只能发现格式和一致性风险，不能判断一个论点在具体赛场上是否成立。
- 不同赛事对质询、自由辩、证据引用和稿件长度的要求不同，请在开局时提供赛制。
