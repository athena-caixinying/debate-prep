# 证据与论证核查

## 证据台账

每条外部证据记录：

```text
E-ID：
要证明的命题：
来源与链接：
发布机构/作者：
日期：
样本与对象：
测量口径：
结论允许达到的强度：
与本题的相关性：
可能被攻击处：
状态：已核验/待核验/仅作线索
```

优先级通常为：原始法律政策与官方统计、同行评审原始研究、权威机构报告、可靠新闻的事实报道、二手评论、案例与轶事。来源权威不等于推论有效，仍需检查样本、口径和相关性。

## 数据检查

- 时间：数据是否适用于辩题指定时期？
- 空间：其他国家、地区或群体能否外推？
- 样本：抽样是否代表讨论主体？
- 指标：测量的是否真是需要证明的概念？
- 基线：比例高低是否有历史或横向参照？
- 因果：相关性是否被误写成因果？
- 量级：即使方向成立，影响是否足以改变胜负？

所有未查源的精确数字标记 `【待核验】`。案例可用于说明机制或反例，不能单独证明普遍趋势。

## 论证链检查

把论证改写为：事实前提 → 机制 → 中间结果 → 最终影响 → 评价标准 → 辩题结论。

逐环检查：

- 前提是否真实且合题？
- 机制是否存在跳步或反向因果？
- 中间结果能否稳定发生？
- 影响是否覆盖题目主体？
- 影响是否足够大、足够持久？
- 从事实到“应该/有错/更好”是否补足规范桥梁？
- 比较辩是否真正比较了对方最强版本？

常见问题包括以偏概全、诉诸完美、滑坡、强加因果、因果倒置、稻草人、偷换概念、虚假两难、循环论证、幸存者偏差和把定义写成结论。使用名称只是为了定位问题，必须解释具体哪一步不成立。

## 表达规则

采用“准确复述 → 承认合理部分 → 指出不能推出之处 → 回到比较标准”的温和反驳结构。避免用“荒谬”“毒瘤”“显然”等强烈词语替代证明。需要强结时，说明对方缺失的前提或证据，而不是只宣布其失败。
